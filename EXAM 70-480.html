    Css Transitions

    In order to understand transitions we must first understand transformations. transformations are the actions we can impose on the elements of the page to move them up down, rotate, scale they are listed below

    • scale(1.2) - must pass a number 1 = 100%
    • rotate(45deg) - will rotate the element on the page, must pass a number and follow with deg
    • rotateX(45deg) and rotateY(45deg) - will rotate the element on the x and y axis (3D effect)
    • translate(12px,12px) - will move the element the number of pixels translate(x,y)
    • skewX(12deg,12deg) - will turn the element to the given angle the number of deg skew(x,y)

    These transforms are applied to certain elements or elements states like div:hover to cause an effect. In order to transition you must add the transition property as well as the transform

    The transition will be passed the 'property'(can be width, translate, all, ect.) to apply the transition to, the 'duration'(must be in seconds '3s'), the 'transition timing function'(I list them below), and 'delay'(must be in seconds '3s')

    • linear - constant speed
    • ease - slow -> fast -> slow
    • easi-in - start slow
    • ease-out - end slow
    • ease-in-out - starts and ends slowly
    • cubic-bezier - ** (p0,p1,p2,p3) define specific values for transition by drawing a cubic bezier curve 
    		div.transitionExample{
			height: 100px;
			width: 200px;
			background-color: red;
			position: absolute;
			transition: all 2s ease-in-out;
    		-webkit-transition: all 2s ease-in-out; /** Chrome & Safari **/
    		-moz-transition: all 2s ease-in-out; /** Firefox **/
    		-o-transition: all 2s ease-in-out; /** Opera **/
		}

		div.transitionExample:hover{
			transform: translate(10px,0px) scale(1.3) rotate(45deg);

    App Cache

    This allows applications to be cached and accessible without internet connections. To enable it you must include manifest attribute in the html tag. You will need to include a Manifest File to specify what needs to be cached

    <html manifest='manifestFileName.appcache'>

    The manifest file has three sections listed below

    • CACHE MANIFEST - list Files to Cache
    • NETWORK - List Files that should never be cached
    • FALLBACK - The pages to serve if a resource is unavailable

    *note that once an application is cached it remains cached until the users clears cache, manifest file is modified, the application cache is programatically updated

    Geolocation

    The object for Geolocation is the navigator.geolocation and the main functions would be the:

    • getCurrentPosition() - returns an object an object with position data

  • Implement a callback

    Receive messages from the HTML5 WebSocket API; use jQuery to make an AJAX call; wire up an event; implement a callback by using anonymous functions; handle the “this” pointer

    • Web Socket

    The WebSocket specification defines an API establishing "socket" connections between a web browser and a server.

    var connection = new WebSocket('ws://address.serversidesocket.com/app',['soap','xmpp'])

    We need to open up a socket connection with the address of the server, notice the Url for the protocol begins with is 'ws' for websocket and 'wss' for secure web socket. The second argument are the optional subprotocols to use.

    We will have three different functions that will run depending on what comes back from the server.
    when the connection is open, we will send some data to the server
    connection.onopen = function(){ connection.send('DataString'); };

    When we get an error back from the Server
    connection.onerror = function(err){ console.log(err); };

    Do something with the message from the server
    connection.onmessage = function(e){ console.log(e.data); };

    We can also send data to the server in our code
    connection.send('Your message');

    Web Worker

    Dedicated Web Workers provide a simple means for web content to run scripts in background threads. Once created, a worker can send messages to the spawning task by posting messages to an event handler specified by the creator.

    You begin by writing the 'WebWorker' process in a separate .js file. Once you create the Webworker, you will be able to post messages between both threads to send information back and forth.

    main.js var myWorker = new Worker("webworker.js");
    myWorker.onmessage = function(e) { console.log("message from the worker" + e.data); }
    myWorker.postMessage("ali"); // the post message function will send data to the webworker

    webworker.js postMessage("This is a message sent to the main.js file");
    onmessage = function(e) { postMessage(e.data) }; // run function when you recieve message from main.js

    myWorker.terminate() to terminate the worker
